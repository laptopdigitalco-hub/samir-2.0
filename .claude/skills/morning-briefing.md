# Morning Briefing

Run the daily morning briefing for fttah. Follow MB-001 exactly.

## Steps

1. Read `intel/focus.md`. Extract: top priority, active deadlines, flagged items. Hold the context — do not report yet.

2. Ask both questions in a single message. Wait for answers before continuing.

```
Two quick questions before your briefing:

1. What did you get done yesterday?
2. What's the one thing you're worried about today?
```

3. Once both are answered, write the briefing using this exact format:

```
# Morning Briefing — [YYYY-MM-DD]

## Today's Focus
[One sentence — the single most important priority from intel/focus.md]

## Yesterday
[One line — what fttah reported getting done. No cheerleading.]

## On the Worry
[Short paragraph reframing the worry. Name it clearly, shift the angle. If solvable, name the lever. If not, name what fttah can control.]

## Question for Today
[One question drawn from the tension between the worry and the priority. Not an action item.]
```

Voice rules: no motivational language, no "you've got this", honest reframe, specific question, every sentence earns its place.

4. Save to `briefings/YYYY-MM-DD.md` using today's date. Confirm: "Briefing saved to briefings/[date].md."

## Failure handling

- `intel/focus.md` missing → ask fttah what they're focused on this week, then proceed
- Only one question answered → ask for the missing answer before continuing
- Save fails → output briefing in chat, note the failure
